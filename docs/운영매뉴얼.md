# Lon — 운영 매뉴얼

> AI 사업제안서 자동 생성기 (FastAPI + Vite/React + MariaDB + MongoDB)
> 본 매뉴얼은 **개발환경 구성 → 일상 운영 → 기동 테스트 → 트러블슈팅** 까지 한 권으로 정리합니다.

---

## 1. 시스템 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | Lon — AI 사업제안서 자동 생성기 |
| 저장소 | https://github.com/zeldahub/autoProposal |
| 백엔드 | FastAPI (Python 3.12) + uvicorn |
| 프론트 | Vite + React 18 + TypeScript + TailwindCSS |
| RDB | MariaDB 11.4 (utf8mb4_unicode_ci) |
| Document DB | MongoDB 8.x |
| 인증 | JWT (HS256, 60분) |
| 비밀키 보관 | AES-256-GCM (BYOK + 암호화 저장) |
| 잡 스케줄러 | APScheduler (in-memory deque) |
| 배포 모드 | (a) 로컬 venv + npm dev / (b) Docker Compose |

### 주요 기능 (Step 1~20 완료)

1. 사업 CRUD + 휴지통 + 복제(템플릿화)
2. AI 키 BYOK (provider 별 활성 키 자동 사용)
3. 첨부 분석(PDF/DOCX/TXT/MD) + LLM 필드 자동 추출
4. PPTX 제안서 / XLSX WBS 자동 생성
5. **PPTX/XLSX 인라인 편집** (다음 version 으로 새 파일)
6. 산출물 미리보기 + 라이브러리 + 다운로드
7. **다국어 지원 (ko/en)** — locale 별 카테고리 응답
8. **사업 협업** — 공유(READ/EDIT) + 댓글 + 알림
9. **데이터 백업** — 사업/사용자 단위 zip
10. 관리자 콘솔 — 사용자/감사로그/카테고리/잡
11. 인앱 알림 (생성/잡/시스템/협업)

### 3-3. MariaDB 초기화

마이그레이션 5종을 순서대로 적용:

```powershell
cd D:\github\autoProposal\apps\api
.\.venv\Scripts\python.exe -c "
from sqlalchemy import create_engine, text
from app.core.config import settings
e = create_engine(settings.mariadb_url)
import re
for v in ['V1__init.sql','V2__seed_categories.sql','V3__notifications.sql']:
    with open(f'D:/github/autoProposal/db/mariadb/migration/{v}','r',encoding='utf-8') as f:
        sql = f.read()
    parts = re.split(r';\s*\n', sql)
    for s in parts:
        s = s.strip()
        if s and not s.startswith('--'):
            try:
                with e.begin() as c:
                    c.execute(text(s))
            except Exception as ex:
                print('skip:', s[:50], '|', str(ex)[:80])
    print('applied', v)
"
```

V4 (i18n), V5 (협업) 는 단일 ALTER/CREATE 문 형태가 깔끔하므로 직접 실행:

## 4. 일상 운영 (PowerShell 스크립트)

`scripts/` 폴더의 4개 스크립트로 일상 운영을 자동화합니다.

### 4-1. 서비스 기동

```powershell
cd D:\github\autoProposal

# (a) 별도 PowerShell 창 2개로 기동 — 로그를 눈으로 확인할 때
.\scripts\start.ps1

# (b) 같은 셸에서 백그라운드 기동 — IDE 통합 작업 시
.\scripts\start.ps1 -Background
```

기동 절차:
1. 8089/5173 점유 프로세스 자동 정리
2. MariaDB(3306) / MongoDB(27017) 응답 점검 (실패해도 계속, 경고만 표시)
3. uvicorn 8089 + vite 5173 기동
4. 헬스체크 (/healthz, /) 결과 출력

### 4-2. 서비스 종료

```powershell
.\scripts\stop.ps1
```
8089/5173 의 모든 OwningProcess 강제 종료 + `.lon-pids` 정리.

### 4-3. 상태 확인

```powershell
.\scripts\status.ps1
```
포트 사용 PID/프로세스명, /healthz 응답 코드, git 브랜치/uncommitted/마지막 커밋을 한눈에.

### 4-4. 배포 (커밋 + 푸시 + PR 머지)

```powershell
# main 직접 푸시 (가장 단순)
.\scripts\deploy.ps1

# 메시지 지정
.\scripts\deploy.ps1 -Message "fix: 알림 레이아웃"

# 새 브랜치 + PR 자동 생성/머지 (squash)
.\scripts\deploy.ps1 -Branch feat/edit-toolbar -Message "feat: 편집 툴바"

# DryRun (실제 변경 없이 변경사항 보기)
.\scripts\deploy.ps1 -DryRun
```

요구사항: `gh auth login` 으로 zeldahub 계정 로그인. 토큰 scope 는 `repo` 필요.

## 6. 기능별 사용법

### 6-1. 로그인 / 회원가입

http://localhost:5173 → 로그인 모드 또는 회원가입 모드 선택.

기본 테스트 계정:

| 이메일 | 비밀번호 | 권한 |
|---|---|---|
| admin@example.com | admin1234 | ADMIN |
| smoke@example.com | secret123 | ADMIN (smoke 테스트용) |

> ❗ `.local` `.test` 등 reserved TLD 는 거부됩니다. `@example.com`, `@gmail.com`, 회사 도메인을 사용하세요.

### 6-2. AI 키 등록 (BYOK)

`/settings/ai` 에서 OpenAI / Gemini / Anthropic 중 하나의 API 키를 등록합니다.

- 키는 AES-256-GCM 으로 암호화 저장 (cipher / iv / tag 분리)
- 응답에는 mask 처리된 미리보기만 포함, 평문 키는 절대 노출되지 않음
- 활성화 토글 → 이후 분석/생성 호출 시 자동 사용
- 테스트 버튼 → echo 호출로 키 검증

### 6-3. 사업 생성 + 분석 + 산출물

`/generator` 에서 한번에:

1. AI 선택 + 키 (또는 저장된 활성 키 사용)
2. 첨부 업로드 — 공고문(NOTICE) + 참조자료(REFERENCE)
3. **분석** 클릭 → LLM 으로 사업 메타 자동 추출 (projectName/goal/scope/...)
4. 사업 정보 수정/추가 입력
5. **생성** — PPTX 제안서 또는 XLSX WBS 즉시 다운로드

생성된 산출물은 `/artifacts` 라이브러리에서도 확인/관리.

### 6-4. 산출물 인라인 편집 (Step 15)

`/artifacts` 또는 사업 상세 산출물 탭 → 미리보기 → **편집** 버튼:

- PPTX: 슬라이드별 title/bullets/speakerNote 직접 수정
- XLSX: 시트 탭 + 셀별 inline input
- **새 버전 저장** → vN+1 으로 새 파일 생성, 원본 보존

### 6-5. 사업 협업 (Step 17)

사업 상세 → **협업** 탭:

- **공유**: 이메일 + 권한(READ/EDIT) 추가, 변경, 해제
- **댓글**: 작성/삭제 (소유자에게 자동 알림 발송)
- 공유받은 사업은 backend `/api/shared-projects` 로 조회 (FE 페이지 추가 예정)

### 6-6. 다국어 (Step 16)

헤더 우측 **KO / EN** 토글로 즉시 전환. localStorage 에 저장. 카테고리 명/시스템 프롬프트는 backend 가 locale 을 보고 응답.

### 6-7. 백업 (Step 18)

- **사업 단일**: 사업 상세 → 헤더의 **백업** 버튼 → `lon-{사업명}.zip`
- **전체**: `/profile` 페이지의 **데이터 백업** 카드 → **전체 백업 (zip)**

zip 구성:
```
manifest.json
projects/<uuid>/
  ├── project.json
  ├── attachments.json + attachments/<file>
  ├── artifacts.json + artifacts/<file>
  ├── comments.json
  ├── analysis/latest.json
  └── llm-sessions.jsonl
```

### 6-8. 알림

헤더의 종 아이콘 → 미확인 알림 (30초마다 폴링).

- GENERATE — PPTX/XLSX 생성 완료
- JOB — 백그라운드 잡 실패 (관리자 전체)
- PROJECT — 협업/공유/댓글
- SYSTEM — 시스템 안내

`/notifications` 에서 전체 알림 + 일괄 읽음/삭제.

### 6-9. 관리자 콘솔 (`/admin`, ADMIN 권한)

| 메뉴 | 기능 |
|---|---|
| 대시보드 | 사용자/사업/산출물/LLM 호출 수 |
| 사용자 | 검색 + role 변경 (USER↔ADMIN; 본인 강등 차단) |
| 감사 로그 | action 필터 + 페이징 |
| 카테고리 | 표준 목차 마스터 (ko/en + 시스템 프롬프트) |
| 잡 | APScheduler 즉시 실행/일시정지/재개 + 최근 20건 이력 |

### 6-10. 사업 휴지통 (Step 12)

- 사업 삭제 = 논리 삭제 → `/projects/trash`
- 휴지통: **복구** 또는 **영구삭제** (cascade — 첨부/산출물/Mongo/LLM 로그 모두)

### 6-11. 사업 복제 (Step 14)

사업 상세 → **복제** 버튼 → 새 이름 + 첨부 포함 여부 선택 → 사업 메타/첨부/Mongo documents 깊은 복사. 산출물은 복제 안 됨 (재생성 의도).

| ADMIN 메뉴 | 대시보드/사용자/감사/카테고리/잡 노출 |

## 8. 트러블슈팅

### 8-1. "세션이 만료되었습니다" 토스트 + 로그인 화면 반복

원인 후보:
- (해결됨) Vite proxy 가 잘못된 포트(8080) → 8089 로 수정 완료
- (해결됨) AuthProvider race condition → finishAuth 동기화로 해결
- 토큰 만료 (60분) — 정상 동작

복구: 다시 로그인.

### 8-2. 로그인 시 "Request failed with status code 500"

원인: Vite dev proxy 가 API에 도달 실패 (ECONNREFUSED).
조치: API 서버가 8089 에서 응답 중인지 확인 → `.\scripts\status.ps1` → API 미가동이면 `.\scripts\start.ps1`.

### 8-3. 422 — "value is not a valid email address ... reserved name"

원인: pydantic `EmailStr` 가 `.local` 등 reserved TLD 거부.
조치: `@example.com` 등 일반 TLD 사용.

### 8-4. 포트 8089 / 5173 이 이미 사용 중

```powershell
.\scripts\stop.ps1
# 또는 직접
Get-NetTCPConnection -LocalPort 8089 | Stop-Process -Force
```

### 8-5. bcrypt 호환 오류

passlib 와 bcrypt 5.x 의 incompatibility. 직접 bcrypt 사용으로 회피되어 있음.
의존성 재설치 시:
```powershell
.\.venv\Scripts\pip install "bcrypt>=4.1"
```

### 8-6. MariaDB ngram parser 미지원 / NULLS LAST

MariaDB 11.4 에는 일부 PostgreSQL 문법이 없음. 본 프로젝트 코드는:
- 검색은 LIKE 사용 (full-text 미사용)
- `ORDER BY ... NULLS LAST` 대신 `Column.is_(None).asc()` + `desc()` 조합

### 8-7. Windows cp949 콘솔에서 한글 깨짐

```powershell
$env:PYTHONIOENCODING = "utf-8"
```
스모크 스크립트는 자체적으로 stdout 을 utf-8 wrapper 로 감쌈.

### 8-8. 한글 파일명 다운로드

Content-Disposition 은 RFC 5987 (`filename*=UTF-8''...`) 사용. 모던 브라우저(Chrome/Edge/Firefox) 자동 처리.

### 8-9. 첨부 분석 시 LLM 미사용 → 휴리스틱 대체

API 키 미등록/비활성/오류 시 placeholder 모드로 진행. PPTX 슬라이드에 `(LLM 미사용 — 수동 보완 필요)` 표시.

### 8-10. APScheduler 잡 누락 메시지 (Run time was missed)

PC 절전/Sleep 후 재개 시 발생. 무시해도 무방. 즉시 실행이 필요하면 관리자 → 잡 → 즉시 실행.
